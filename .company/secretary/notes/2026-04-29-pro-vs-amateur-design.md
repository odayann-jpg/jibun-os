# プロのデザイナーと素人デザインの違い — 直せる粒度の具体ルール集

作成日: 2026-04-29
目的: 自分・チームの制作物（LP・スライド・営業資料・アプリUI・写真選定）を見て「素人っぽさ」を具体的な要素で判別・修正できるようにする。「センス」ではなく**測れる・直せる粒度**で抽出。

---

## A. タイポグラフィ

### 行間（line-height / leading）
- **プロ**: 本文は font-size の **1.4〜1.6倍**（=140〜160%）。見出しは **1.1〜1.3倍**（短いから詰める）。8ptグリッドなら 16px本文に 24px line-height（×1.5）。**4の倍数**にスナップ。
- **素人**: 全テキストに同じ line-height。本文1.2倍以下で窮屈、見出し1.5倍以上で間延び。
- **理由**: 短い見出しは詰めて塊感、長い本文はあけて呼吸を作る。

### 字間（letter-spacing / tracking）
- **プロ**: 大見出し（24px超）は **−1〜−2%**（−10〜−20/1000em）でわずかに詰める。本文は基本0。和文見出しは「文字詰め」で約物の余白を詰める。
- **素人**: デフォルトのまま。大見出しがスカスカ。逆に本文に letter-spacing を入れて読みにくくする。

### フォントサイズの階層（Type Scale）
- **プロ**: 比率を1つ決めて全サイズを生成。**1.125（密度重視）／1.25（標準）／1.333（LP向け）／1.5（インパクト）／1.618（黄金比）**。例：base 16px × 1.25 → 20, 25, 31, 39, 49…
- **素人**: 「14, 15, 17, 22, 28」のような根拠のない散発的な数値。
- **H1→本文の比率**: LPなら **2.5〜3倍以上**でジャンプ率を作る。資料・ダッシュボードなら 1.5〜2倍。

### 行長（measure）
- **プロ**: 1行 **45〜75文字**（英数字）、和文 **全角 25〜40字**。コラム幅で物理的に制限。
- **素人**: 画面幅いっぱいに本文を流す（1行100字超）。改行で目が迷子に。

### ウェイトコントラスト
- **プロ**: 同ファミリー内で **2〜3段階のジャンプ**（Regular 400 → Bold 700）。見出しと本文で大きな差。
- **素人**: 全部Regular、または Medium と Semibold のような近すぎる差で「メリハリなし」。

### 和欧混植・約物処理（日本語特有）
- **プロ**: 句読点・カッコの**半角アキ（50%）を詰める**。和文と欧文のベースラインを揃える。**禁則処理**（行頭に「、。」が来ない、行末に「（」が来ない）。
- **素人**: ブラウザ・PowerPointのデフォルトのまま。約物前後にスペースが空きすぎ。

### スマートクオート
- **プロ**: `"hello"` ではなく `"hello"`（カーリー）。アポストロフィも `'` ではなく `'`。
- **素人**: ストレートクオートのまま → 一目で素人組版とバレる。

## B. レイアウト・グリッド・余白

### 8pt / 4pt グリッド
- **プロ**: padding・margin・サイズを **4または8の倍数**（4, 8, 12, 16, 24, 32, 40, 48, 64, 80）で統一。整数pxにスナップ。
- **素人**: 「13, 17, 22, 35」のような半端な数値が混在。整列していないように見える。

### 内側 ≤ 外側ルール
- **プロ**: 要素の**内側のpaddingは外側のmarginより小さく**する。例：カード内padding 16px、カード間margin 24px。
- **素人**: 内側がスカスカで外側が詰まっている。グルーピングが崩壊。

### CRAP原則（近接・整列・反復・対比）
- **プロ**: 関連要素は近づけ（**近接**）、直線で揃え（**整列**、左揃え原則）、同じスタイルを繰り返し（**反復**）、重要要素を強くする（**対比**）。
- **素人**: 中央揃えを多用、関連要素間の距離がバラバラ、装飾が毎回違う。

### ホワイトスペース
- **プロ**: 余白は**意図的な要素**。重要なものほど余白を多く取る（KV周囲、CTA上下）。
- **素人**: 「もったいない」と情報を詰める。視線の休憩がなく疲れる。

### 視線誘導
- **プロ**: 横長は**Z型**、縦長テキストは**F型**、ポスターは**グーテンベルク（左上→右下）**。重要要素を視線パスの起点・終点に。
- **素人**: 視線が散る配置。CTAが視線のゴールに置かれていない。

## C. 配色

### 60-30-10ルール
- **プロ**: ベース60%／サブ30%／アクセント10%。アクセントは画面の **5〜10%以下**に絞る。
- **素人**: 主役色を画面の30%以上に塗る。CTAが目立たない。

### 純黒・純白を避ける
- **プロ**: 文字は `#000` ではなく **`#1B1B1B` `#222` `#333` `#424242`**。背景白も `#FFF` ではなく `#FAFAFA` `#F7F7F7`。グレーにわずかに**色味（hue）**を足す。
- **素人**: `#000` × `#FFF`。コントラスト過剰で目が疲れ、画面に「重さ」が出る。

### コントラスト比（WCAG）
- **プロ**: 本文は **4.5:1以上（AA）**、できれば **7:1（AAA）**。大きな文字（18pt以上 or 14pt太字）は3:1以上。WebAIM Contrast Checker で検証。
- **素人**: 薄いグレー文字（`#CCC` on white = 1.6:1）を「おしゃれ」と思って使う。読めない。

### 彩度コントロール
- **プロ**: 鮮やかな原色（HSLのS=100%）はほぼ使わない。彩度を **40〜70%**に落とし、明度で階調を作る。
- **素人**: ピュアな赤・青・緑・黄を多用。「子供っぽい」「派手」になる。

### 配色関係
- **プロ**: 1つの色相を選び、**類似色**（隣り合う色）か**補色**（向かい合う色）で構成。
- **素人**: 色を「なんとなく好き」で追加。10色以上のごちゃ混ぜ。

## D. 写真・画像選定

### 視線方向
- **プロ**: 被写体の視線方向に**余白を空ける**（subject looking right → 画面左に置き、右に余白）。
- **素人**: 視線の先を切り取る。閉塞感・居心地の悪さが出る。

### 3分割法
- **プロ**: 画面を縦横3等分し、**4つの交点**のいずれかに主役を置く。ポートレートは**目を上の交点**に。
- **素人**: 主役を中央に置く（証明写真感）。

### 写真同士のトーン統一
- **プロ**: 複数写真の**色温度（暖色/寒色）・彩度・明度**を揃える。Lightroom等で一括補正。
- **素人**: ストック写真をそのまま並べる。1枚は青っぽく、1枚は黄色っぽくバラバラ。

### ストック写真の選び方
- **プロ**: 「いかにもストック」な合成笑顔・握手・電球マークを避ける。**具体的・固有性のある**写真を選ぶ。
- **素人**: 「ビジネス 成功」で出てきた1枚目を使う。LP全体の説得力が落ちる。

### アスペクト比
- **プロ**: コンテンツに合わせて使い分け。**16:9**（動画・ヒーロー）／**4:3 or 3:2**（記事サムネ）／**1:1**（SNS）／**9:16**（モバイル縦）／**3:4**（人物縦）。同じ画面で混在させない。
- **素人**: 元の比率のまま貼り付け、グリッドが崩れる。

### クロップ
- **プロ**: 顔は**おでこの上を切る**（目を上1/3に）、関節（手首・足首・膝）でクロップしない。
- **素人**: 関節で切る、頭頂部に余白がありすぎる。

## E. アイコン・装飾

### スタイル統一
- **プロ**: アイコンセットを**1種類**に固定（線画なら全部線画、塗りなら全部塗り、線幅同一 1.5px or 2px）。
- **素人**: 無料サイトから集めて混ぜる。線画と塗りが混在、線幅もバラバラ。

### サイズ基準
- **プロ**: テキスト隣接アイコンは**本文サイズ × 1.0〜1.25倍**、UI操作のアイコンは **20px or 24px** で統一。
- **素人**: アイコンごとにサイズが違う。

### 引き算
- **プロ**: グラデーション・影・装飾線を**外す勇気**。情報のヒエラルキーで魅せる。
- **素人**: 装飾で埋める。シャドウ・ボーダー・グラデ・ストライプ全部入り。

---

## F. UIコンポーネント

### ボタンpadding
- **プロ**: **水平padding は垂直の 2〜3倍**（例：12px / 24px）。ボタン高さは8の倍数（40 or 48px）。タップターゲットは**44×44px以上**（Apple HIG）。
- **素人**: 正方形に近い（12 / 12）か、padding なしでテキストにボーダー → 「押せる感じがしない」。

### 角丸の文脈統一
- **プロ**: ボタン・カード・入力欄で角丸を**設計（4 / 8 / 12px）して統一**。同じ意味の要素は同じ角丸。
- **素人**: 場所ごとにバラバラ（ボタン6、カード12、画像0）。

### シャドウの一貫性
- **プロ**: 光源は**画面上方**から固定。x=0、y=正の値、blurはspreadの数倍、色は黒ではなく**ブランドカラーを混ぜたダークトーン**＋低opacity（0.08〜0.15）。Elevationシステムで標高ごとにシャドウを定義。
- **素人**: シャドウの方向がバラバラ、`box-shadow: 0 0 10px black` のような濃い灰色シャドウで重い。

### 状態の設計
- **プロ**: Hover / Focus / Active / Disabled / Loading / Empty / Error の **7状態**を全部設計。
- **素人**: デフォルト状態しか作らない。

## G. 細部の隠し技

- **カーニング**: ロゴ・KVの大見出しは1文字ずつ字間を手調整。"AV" "WA" などで詰める
- **スマートクオート・ダッシュ**: `"smart"` `'apostrophe'` `—em dash` `–en dash` `…ellipsis` を使い分け
- **整数pxスナップ**: 13.5px, 17.3px のような小数pxを避ける（ぼやける）。Retinaは2の倍数で
- **ハイフネーション・禁則**: Webは `hyphens: none;` 基本。和文は `word-break: keep-all;` ＋ 禁則処理

## H. 思考プロセスの違い

| 観点 | プロ | 素人 |
|---|---|---|
| 着手 | ブリーフを読み込む（目的・対象・トーン）。グリッド設計から入る | いきなり装飾・色を試す |
| 制約 | 制約を歓迎（ブランドカラー縛り、文字数縛り）。制約が発想を絞る | 自由を求めて発散しすぎる |
| 試作 | 20〜50案のサムネを描く（ラフ・スケッチ）→絞る | 1案を作り込み、ダメだと最初からやり直し |
| フィードバック | 早期に出す（粗い段階で見せる）。修正を「設計の一部」として歓迎 | 完成してから出す。修正されると傷つく |
| 完成判定 | 引き算で終わる（「もう要素を減らせないか？」） | 足し算で終わる（「もっと装飾できないか？」） |
| 整数規律 | グリッド・倍数・比率に規律を持って従う | 「なんとなく」で値を決める |

## まとめ：5秒チェックの観点（経営者用）

制作物を見たとき、まずこの5つを確認すれば「素人っぽさ」の8割は捕まえられる：

1. **数値の規律**: padding・margin・font-size がすべて4または8の倍数か
2. **コントラスト**: 文字が `#000` でないか／背景が `#FFF` でないか／本文が4.5:1以上か
3. **行間と行長**: 本文 line-height 1.5前後、1行 45〜75字（和文25〜40字）
4. **アクセントの占有率**: ブランド主色＋CTA色が画面の10%以下に絞られているか
5. **写真のトーン**: 並んだ写真の色温度・彩度が揃っているか
