## 日報の受け取りと処理フロー

### 日報パースルール

誠司が日報をコピペしたら、以下の手順を実行する:

1. **データ抽出**: 日報から以下の値を読み取る

| 日報の表記 | sales.json のキー |
|-----------|-----------------|
| 本日の出品（スタッフ名 + 点数） | `staff.スタッフ名.listings` |
| 月間累計（スタッフ名 + 点数） | `staff.スタッフ名.cumulative` |
| 月間累計合計 | `monthly_cumulative.staff_listings` |
| ◯買取 + 件数 | `buy_count` |
| ◯買取 + 金額 | `purchase_amount` |
| ◯ネット売上 + 点数 + 金額 | `net_sales`, `net_items` |
| ◯店頭売上 + 点数 + 金額 | `store_sales`, `store_items` |
| 月間売上（ネット） | `monthly_cumulative.net_sales`, `monthly_cumulative.net_items` |
| 月間売上（店頭） | `monthly_cumulative.store_sales` |
| 月間売上（総合） | `monthly_cumulative.total_sales` |
| 業務・所感のテキスト | `highlights`（要約して記録） |
| 日報全文 | `raw_report` |

2. **sales.json を更新**
3. **generate_dashboard.py を実行** してダッシュボードを再生成
4. **集計結果を報告**:
   - 今日の売上・買取サマリー
   - 月累計売上
   - 月次目標達成率（目標が設定されている場合）
   - 前日比・前週比のコメント

## 経費の入力

誠司が「経費入力して」「○○費¥○○」と言ったら:

1. `data/sales.json` の該当月 `expenses` 配列に追加:

```json
{
  "date": "2026-04-15",
  "category": "広告費",
  "amount": 50000,
  "memo": "Instagram広告 4月分"
}
```

2. カテゴリの例: `広告費` / `備品費` / `交通費` / `外注費` / `消耗品費` / `その他`
3. `generate_dashboard.py` を実行してダッシュボードを更新
4. 「¥○○ (カテゴリ名) を記録しました。今月の変動費合計: ¥○○」と報告

## 月次目標の設定

誠司が「今月の目標は○○万円」と言ったら:

該当月の `target.total_sales` を更新する。
ダッシュボードに達成率バーが表示される。

## 月次振り返りの依頼

「先月の振り返りして」「4月まとめて」と言われたら:

1. sales.json の該当月を読む
2. 以下を分析して報告:

```

【売上サマリー】
売上合計: ¥○○（目標比 ○%）
- ネット: ¥○○（○点）
- 店頭: ¥○○（○点）
日販平均: ¥○○
1点単価: ¥○○

【買取】
買取件数: ○件
買取金額: ¥○○
買取単価: ¥○○/件

【スタッフ出品】
菊池: ○点 / 島野: ○点

【P/L】
粗利: ¥○○（粗利率 ○%）
固定費: ¥○○
変動費: ¥○○
営業利益: ¥○○

【前月比】
売上: ±○% / 粗利率: ±○pt

【コメント・提案】
（Claude が気づいたことを添える）
```

## 週次振り返りの依頼

「今週の振り返り」「第○週のまとめ」と言われたら:

該当週（日〜土）の日次データを集計して報告。
前週比、今月ペースの着地予想も添える。

## ダッシュボードの更新

sales.json を更新したら必ず:

```bash
python3 generate_dashboard.py
```

を実行してダッシュボードを再生成する。

## 注意事項

- sales.json はプライベートデータ。git には含まれない（.gitignore で除外済み）
- 同じ日に日報が届いたら上書き更新してOK
- 「スタッフ名」の表記は `菊池` か `島野` に統一（日報の表記ゆれを吸収すること）
  - 「菊地」「菊池」など揺れがあっても `菊池` として記録
- 未決済の追跡は不要（ベクトル管理）
