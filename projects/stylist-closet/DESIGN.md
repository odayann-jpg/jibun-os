## Typography (現状)

```
font-family: 'Hiragino Sans', 'Yu Gothic', sans-serif
base:        14px

/* 階層（散らばっている）*/
11px: ラベル、テーブルヘッダ、バッジ
12px: メタ情報、日次テーブル
13px: ボタン、テーブル本文
14px: body
16px: ヘッダ h1
```

**現状の問題**: スケール比が均一でない（1.083倍, 1.083倍, 1.077倍, 1.143倍, 1.375倍）。h1 が 16px と小さく、カード数字 22px が「最大」になっている逆転がある。weight contrast も弱め。

## Radius (現状)

## Components (現状)

- **ヘッダ**: ネイビー塗りつぶし、白文字、sticky
- **月ナビ**: ピル型ボタン横並び、active はネイビー塗り
- **タブナビ**: 下線アクティブ、テキスト系
- **カード**: 白背景＋薄ボーダー＋小さな影、`label / value / sub` 構造、`green/red/blue` 色クラスで強調
- **パネル**: カード同サイズ感、`h3` は uppercase 11px、下線あり
- **テーブル**: `total-row`（強調行）と `section-header`（タイトル行）あり、ホバー背景
- **進捗バー**: 細い、達成率バー
- **バッジ**: 土日マーク（`badge-sat`, `badge-sun`）、カテゴリ
- **チャート**: Chart.js v4、`max-height: 180px` で抑えめ

## Motion (現状)

- ボタン `transition: all 0.2s` 多用
- 進捗バー `transition: width 0.5s`
- イージング指定なし（全部 default = ease）

**現状の問題**: `all` トランジション、ease-out 系の指定なし。

## 反すべきパターン（現状で起きていそうなもの）

- 絵文字をタブと h1 に使っている（📊 📅 🛍 👥 📦）→ プロフェッショナルなトーンとは噛み合いが弱い、要見直し
- `box-shadow: rgba(0,0,0,...)` → ブランド色にティントすべき
- `background: #f8f8f8` などハードコード残骸あり
- `text-transform: uppercase` を和文混在ヘッダで使っている → 和文には効かないので装飾的にも意味薄い

## polish で目指す方向（提案）

1. トークンを OKLCH ベースに再定義し、ブランド色（ネイビー）を「ティントの軸」として全ニュートラルに薄く効かせる
2. タイポスケールを 1.25 倍ベースに整え、weight contrast を効かせる（h1 を主役に戻す）
3. スペーシングを 4px ベースに統一（4 / 8 / 12 / 16 / 24 / 32）
4. 絵文字依存をやめ、必要ならアイコンか、見出しの weight・サイズで階層を作る
5. uppercase 和文をやめる（和文の階層は字間・weight・色で作る）
6. ヘッダのネイビー塗り＋白文字を見直し、より静かに（西岡が「サンプル感」を感じない品位に）
7. 影とボーダーをブランド色のティントに統一
8. モーション指定を ease-out-quart に統一、 `transition: all` を廃止
