## 3. デザイン原則（最重要・絶対）

> **装飾ではなく、構造とロジックで伝える。**
> **タイポグラフィとレイアウトの綺麗さで、上質さを出す。**

これがすべての判断の上位ルール。色・装飾・モーション・コピーで迷ったら、ここに戻る。

---

## 4. Anti-references（絶対に避ける）

- 派手なグラデーション
- 安っぽいスクールチラシのような雰囲気
- 昔の情報商材のような文字の羅列
- 強い言葉が並んでいる胡散臭いページ
- ベージュ背景（白ベース固定）
- 過剰な絵文字・装飾線・装飾的な囲み
- 不必要な影・グロー・パララックス
- バウンス・エラスティック・派手なスクロールアニメ

## 7. 磨いてほしい箇所（ここを集中的に押す）

設計の方向性は固まっているので、**以下の限定的な領域**だけ磨いてください。

### 7.1 視覚的なリズム
14 セクションが続くと、似たパターンの繰り返しで単調に感じる可能性がある。
- セクション間の「呼吸」を変える方法（一部のセクションだけ余白を倍にする、罫線の長さを変える等）
- 章の役割（前提を語る章 / リストを並べる章 / 物語を語る章）に応じた視覚的差異
- 「装飾を足す」のではなく「構造のリズム」で変化を作る

### 7.2 Hero の typographic 強度
現在は 32px Mincho の素直な組み方。「夜の書斎で開いた最初の一秒」で**静かに引き込む**力をもう一段。
- 行送り、letter-spacing、quote 文字 (『』) のサイズ・色の扱い
- Hero 上部の eyebrow キャプションの存在感
- 32px → 30px or 34px の検証（mobile も含めて）

### 7.3 引用・自問のブロック
Section 02 にある「『もっと稼いでから』」「『自分は、ちゃんと相手を幸せにできるのだろうか』」など、感情の核に近い引用箇所。
- 現状は 2px navy 左ボーダーの統一感
- もう少し「自問の重み」を出す treatment（例: より静かな intent、空白の使い方、フォントサイズの差別化）

### 7.4 Profile セクション
立花さんの写真（円形 200px）が後から入る。**写真の周囲のテキスト組み版**をもっと印象的に。
- 名前 / 英名 / 肩書 / リード文 の階層
- 経歴の credentials 表（登壇実績・専門領域・経歴）の組み方
- 写真と本文の関係（重なり・余白・並び方）

### 7.5 CTA の treatment
現状はアウトライン四角ボタン。「これはいいな」の納得から自然に問い合わせへ繋ぐ、もっと **静かで品のある CTA**。
- 派手なボタンは禁止
- アンダーライン処理 / 細い罫線 / 章として独立させる、などの探求

### 7.6 微細なモーション（任意・最小限）
現状はモーションゼロ。**読む邪魔をしない範囲で**、エントランスに 1 つだけ加えるなら？
- フェードイン 200ms ease-out-quart 程度の最小限
- `prefers-reduced-motion` 対応必須
- 過剰なら入れないほうが良い

### 7.7 Mobile タイポの最終調整
mobile 375px で読みやすさを最大化。
- Body 14px / line 2.05 はちょうど良いか
- Hero 24px / line 1.8 は適切か
- Section 間隔 72px は適切か

## 8. 絶対に守るべき制約（破ったら NG）

1. **コピーは1文字も変えない**。誤字修正であっても勝手にいじらない（必要なら指摘して質問する）
2. **カラーパレットは追加しない**。Paper / Ink / Body / Navy / Navy Soft / Line のみ
3. **新しいフォントを足さない**。Hiragino Mincho / Hiragino Gothic / EB Garamond だけ
4. **14 ブロック構成は変えない**。順番・統合・分割しない
5. **白以外の背景色を使わない**。一切のベージュ・薄ベージュ・薄黄ベージュ禁止
6. **装飾的なアイコン・絵文字・SVGイラストを足さない**。typographic マーク（✓ × ◆ ◇ ■ → ・）のみ
7. **静的 HTML+CSS のまま**。React / Vue / フレームワーク化しない
8. **画像追加は最小限**。コンテンツの主役は文字
9. **CSS-in-JS / Tailwind に書き換えない**。既存の `<style>` ブロック構造を維持
10. **アクセシビリティを後退させない**：focus / contrast / semantic / `lang="ja"`

## 9. 参考にしてほしいデザインの感触

- **Aesop**（aesop.com）— 静謐な明朝＋ローマンの混植、白ベース、極限まで削ぐ
- **Aperture / Kinfolk**（写真集・雑誌）— エディトリアルの章立て、italic 副題、階層
- **暮しの手帖** — 日本の上質編集物の余韻、押しつけない知性
- **FT Weekend / Monocle** — 新聞・雑誌組版の知的な秩序
- **松浦弥太郎の本のカバーや内側組版** — 静かで余白がある構成

**避けたい参考**：
- ❌ Apple product page（プロダクト写真主体すぎる）
- ❌ Stripe / Linear（テック過ぎる）
- ❌ Awwwards 系の演出が強いLP（演出主体）
